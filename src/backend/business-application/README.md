# Business Application Service

The **Business Application** is the core backend microservice of the LMS platform. It manages course content, user directories, quizzes, coding problems, submissions, and interview sessions. It relies on the **Auth Provider** service for JWT key loading and authentication verification.

---

## Technologies Used

- **Python 3.14+**
- **FastAPI**: Web framework for building APIs.
- **SQLAlchemy (Async)**: SQL toolkit and Object Relational Mapper (ORM) using asynchronous operations.
- **Alembic**: Database migrations management.
- **Asyncpg**: Fast PostgreSQL database client library for Python.
- **UV**: Fast Python package and environment manager.
- **Uvicorn**: ASGI web server implementation.

---

## Getting Started & Setup

Follow these steps to configure, migrate, and run the Business Application service locally.

### Prerequisites

Ensure the following are installed:
- [UV Package Manager](https://docs.astral.sh/uv/getting-started/installation/): Packing Manager using for this project. 
- Docker (with the root infrastructure running)

---

### Step 1: Initialize the Environment

1. Navigate to the service directory:
   ```bash
   cd src/backend/business-application
   ```

2. Create a virtual environment and install all dependencies:
   ```bash
   uv sync
   ```
   *(This reads `pyproject.toml` and locks in `uv.lock`, automatically setting up the `.venv` folder).*

---

### Step 2: Setup Environment Variables

1. Copy the example environment file:
   ```bash
   cp .env.example .env
   ```

2. Open `.env` and adjust the variables if necessary. The default variables are set to work out of the box with the default root Docker compose configurations:
   ```env
   DATABASE_URL=postgresql+asyncpg://lms:lms@localhost:5432/lms
   VERIFY_REGISTER_SECRET=any-long-random-string-used-for-validation
   AUTH_PROVIDER_URL=http://localhost:4001
   ```
To generate a secret key, you can paste any uuid string generated by this website: https://www.uuidgenerator.net/version4
---

### Step 3: Run Database Migrations

Before launching the server, run the database migrations using Alembic to create the tables in your PostgreSQL database instance:

```bash
uv run alembic upgrade head
```

---

### Step 4: Run the Service

Make sure:
1. Your root Docker infrastructure (PostgreSQL, Redis, RabbitMQ) is up and running.
2. The **Auth Provider** service is running (on port `4001`) since this application queries it on startup to fetch the JWT public key.

Start the FastAPI application:
```bash
uv run main.py
```
*(The server starts Uvicorn, serving the API on [http://localhost:4000](http://localhost:4000) with hot-reload enabled).*

---

## API Endpoints

Once running, you can explore the interactive API docs:
- **Swagger UI**: [http://localhost:4000/docs](http://localhost:4000/docs)

### Key Endpoints

- `GET /api/v1/health` - Check health status of the database and application.
- `GET /api/v1/auth/profile` - Retrieve profiles for authenticated users.
- Various CRUD endpoints for quizzes, questions, tags, and interview sessions.
